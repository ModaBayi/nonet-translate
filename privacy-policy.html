Privacy Policy

Last updated: May 17, 2026

This Privacy Policy describes how NoNet Translate handles information in connection with the NoNet Translate mobile application. This document is intended to support the app's public listing and user transparency requirements.

1. Scope

This policy applies to the NoNet Translate mobile application, this public project website, and related user support communications.

2. Developer identity

Application name: NoNet Translate
Public project repository: github.com/ModaBayi/nonet-translate

Before submitting to Google Play, add your preferred support email address to both this policy and your store listing so the public contact details match.

3. Data handling summary

NoNet Translate is intended to minimize data exposure and provide a straightforward translation experience. Review this page carefully before publishing the app to make sure it accurately reflects the current app build.

  • The app should only request permissions that are necessary for visible user-facing features.
  • If the app processes text entered by the user, that processing should be limited to providing the requested translation feature.
  • If analytics, crash reporting, advertising, cloud backup, or account systems are added later, this policy must be updated before release.

4. Information the app may process

Depending on the final feature set, the app may process user-provided text, language preferences, and basic app settings required to operate translation features.

If a released build collects or transmits any additional categories of personal or sensitive data, those categories must be explicitly disclosed here and in Google Play's Data safety section.

5. Data sharing

The intended baseline for NoNet Translate is not to sell user data. If any data is shared with service providers such as hosting, analytics, crash reporting, advertising, or translation infrastructure providers, that sharing must be transparently disclosed before release.

6. Security, retention, and deletion

  • Data should be stored only as long as necessary for the app feature being used.
  • Where possible, data should be processed locally or minimized before transmission.
  • If the app later introduces user accounts, account deletion and associated data deletion instructions must be added here and in the app.

7. Children

NoNet Translate is not presented as a child-directed application in this public draft. If you decide to target children or mixed-age audiences, you must revisit the Play Console target audience, ads, and Families Policy requirements before publishing.

8. Changes to this policy

This policy may be updated from time to time to reflect changes to the application, legal requirements, or support practices. Material updates should be published on this page.

9. Contact

Until a dedicated support email is added, the public support page for this project is available at support.html.
